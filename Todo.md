# Dashboard Editorial Redesign Skill

## Objective

Transform an existing dashboard into a premium, handcrafted interface that feels intentionally designed by an experienced product designer rather than generated by AI.

This skill should create dashboards comparable to products released in **2026**, emphasizing elegance, usability, and consistency instead of visual complexity.

This skill **must preserve all existing functionality** while elevating the visual language.

---

# REQUIRED FIRST STEP

Before performing any analysis or implementation, you MUST locate and read the following document from the project root:

```
premium-editorial-ui.md
```

This document is considered the project's primary design language.

It defines:

- typography
- spacing
- color palette
- design philosophy
- component styling
- interaction principles
- visual identity

This dashboard MUST inherit and extend that visual language.

Never redesign the dashboard using a different design style.

---

# REQUIRED SECOND STEP

Do NOT immediately modify the code.

Instead, perform a complete design review.

Inspect:

- existing layout
- navigation
- user flows
- information hierarchy
- dashboard widgets
- component consistency
- responsive behavior
- accessibility
- typography
- spacing
- color usage

Then summarize your findings.

---

# REQUIRED THIRD STEP

Before implementation, ask the user for clarification.

Never assume design preferences.

Clarify items such as:

- preferred dashboard density
- desired sidebar behavior
- preferred widget layout
- animation intensity
- chart preferences
- card style
- data prioritization
- visual emphasis
- mobile expectations

Example:

> I have completed the design analysis and reviewed the Premium Editorial UI guide.
>
> Before implementing the redesign, I'd like to confirm a few preferences:
>
> - Should the dashboard feel more minimal or data-dense?
> - Would you like a fixed or collapsible sidebar?
> - Should charts be emphasized or minimized?
> - Should cards remain rectangular or become softer and more editorial?
> - Would you like subtle motion or almost no animations?

Only begin implementation after receiving clarification.

---

# Primary Design Goal

The dashboard should feel like it belongs to the same product as the landing page.

Users should immediately recognize that both pages belong to one coherent design system.

Never create a dashboard that appears visually disconnected from the welcome page.

---

# Design Philosophy

The dashboard should feel:

- calm
- elegant
- premium
- handcrafted
- timeless
- sophisticated
- trustworthy
- focused

Never create a dashboard that feels:

- template-based
- corporate
- Bootstrap-like
- generic SaaS
- AI generated
- overly colorful
- cluttered
